Phase 3
Add Anthropic Key to Thumbmaker
5 min — one copy-paste in a text editor
10
You
Open LemonberryX_Thumbmaker.html in Notepad/TextEdit — NOT your browser
Right-click the file → Open With → Notepad (Windows) or TextEdit (Mac).
Mac only: if TextEdit opens → go to Format menu → click Make Plain Text before touching anything.
11
You
Press Ctrl+F (Cmd+F on Mac) and search: api.anthropic.com
You'll land on a line starting with const res = await fetch. This is the line you need.
12
You
Find 'Content-Type':'application/json' and replace that whole piece
Find: 'Content-Type':'application/json'

Replace with this (swap YOUR_KEY_HERE with your actual Anthropic key):
'Content-Type':'application/json','x-api-key':'YOUR_KEY_HERE','anthropic-version':'2023-06-01','anthropic-dangerous-direct-browser-access':'true'
Save (Ctrl+S / Cmd+S).
⚠️ Never share this file after adding the key. It's local only — never uploads anywhere when run from your computer.

Phase 7
Install GPU Video Generation Tools
All typed into your RunPod terminal — type each line, press Enter, wait before typing next
33
GPU
🔴 Flagship — install this first
Install HunyuanVideo-1.5 — 720p in 75 seconds, image-to-video
Type these one at a time in the RunPod terminal:
conda create -n HunyuanVideo python=3.11.9 -y
conda activate HunyuanVideo
conda install pytorch==2.4.0 torchvision pytorch-cuda=12.4 -c pytorch -c nvidia -y
git clone https://github.com/Tencent-Hunyuan/HunyuanVideo-1.5
cd HunyuanVideo-1.5
pip install -r requirements.txt
Takes 20–30 min. Model weights alone are ~13GB. Let it download — don't close the terminal. Go make coffee.
To generate: python sample_video.py --prompt "cinematic vault room gold bars dramatic lighting" --video-size 544 960 --video-length 33 --infer-steps 30

The I2V model (Image-to-Video) is what you'll use most — take a Whisk image of a vault, a ship, a crime scene → turn it into a 5-second cinematic clip.
34
GPU
🔴 Long coherent video — constant memory use (min 6GB)
Install FramePack — long-form AI video with no memory limit
In RunPod terminal (can do this after HunyuanVideo finishes):
git clone https://github.com/lllyasviel/FramePack
cd FramePack
pip install -r requirements.txt
python demo_gradio.py
The last command opens a web UI. In RunPod → Connect → HTTP Service → Port 7860 → opens in browser.
FramePack is unique: it generates frame-by-frame with constant memory — a 2-minute video uses the same VRAM as a 10-second one. This is what you use for long cinematic sequences in Grimm Archives without running out of memory mid-generation.
35
GPU
🟡 Speed-first B-roll (10 seconds per clip) — needs 14GB VRAM
Install LTX-Video — fast B-roll at scale
git clone https://github.com/Lightricks/LTX-Video
cd LTX-Video
pip install -e .
LTX-Video generates in ~10 seconds per clip. Not as cinematic as HunyuanVideo but 8–10x faster. Use for pattern interrupts every 60–90s in your videos — you need a lot of B-roll clips, and LTX cranks them out fast.
